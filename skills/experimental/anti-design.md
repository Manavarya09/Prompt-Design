# Anti-Design / Chaos UI

## Intent
Deliberately challenge design conventions, creating deliberately discordant, unexpected interfaces that question assumptions about what design should be.

## Principles
1. **Rule breaking** - Conventions are meant to be shattered
2. **Intentional dissonance** - Clash is the point
3. **Unexpected placement** - Nothing where it should be
4. **Overwhelming stimuli** - More is more
5. **Deconstruction** - Take apart and reassemble wrong

## Layout Rules
- No grid or deliberately broken grid
- Overlapping elements
- Off-center placement
- Mixed orientations
- Intentionally cluttered sections
- Random positioning
- Full-bleed chaos
- Impossibly complex overlaps

## Typography
- Font: Mix everything (serif + sans + display)
- Sizes: Way too small or way too large
- Random rotation
- Stacked text
- Mixed directions
- Broken alignment
- Layered text
- Decorative overuse

## Colors
- Maximum clashing
- Random color choices
- Saturated, jarring colors
- Patterned backgrounds
- Gradient clashes
- Neon on neon
- Random accent application
- No color harmony

## Components
- Chaotic Buttons: Overlapping, misaligned
- Colliding Cards: Stacked on each other
- Messy Inputs: Multiple overlapping
- Mixed Icons: Inconsistent style
- Pattern Overlays: Textures everywhere
- Broken Navigation: Parts everywhere
- Layered Modals: Stacked chaos
- Conflicting Badges: Clashing colors

## Do's
- Break all layout rules
- Use clashing colors
- Mix font families randomly
- Overlap elements intentionally
- Create visual tension
- Question conventions
- Make bold statements
- Embrace chaos

## Don'ts
- Try to make it look "right"
- Follow conventions
- Create harmony
- Add polish
- Make it accessible (ironically)
- Explain the design
- Apologize for chaos
- Try to please everyone

## Agent Instructions
- Break grid layouts completely
- Overlap elements
- Use clashing colors
- Mix font families randomly
- Create intentional dissonance
- Remove all structure
- Add chaotic elements
- Question design assumptions

## Output Constraints
- HTML/CSS: Unconventional positioning
- React: Chaotic component structure
- Position: Absolute, fixed, or intentionally wrong
- Font: Random or mixed
- Colors: Maximum clash
- Structure: Broken or absent
